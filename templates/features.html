01 / Latency Profiling

Every percentile.
Not just the average.

Averages lie. A model with 5ms average latency can still have 200ms p99 spikes that destroy user experience in production. ModelBench captures the full picture — min, avg, median, p90, p95, p99 and max — so you know exactly what your tail latency looks like before you ship.

  • Configurable warmup runs to exclude cold-start bias from results
  • Per-iteration latency trace for spotting spikes and jitter
  • Histogram with 20 bins to visualise the full latency distribution
  • Standard deviation alongside percentiles for statistical confidence

02 / Memory & CPU

Profile peak heap,
RAM, and CPU together.

A model that looks fast in latency benchmarks can still silently exhaust your server's memory. ModelBench tracks peak heap allocation using Python's tracemalloc, total process RAM via psutil, and CPU utilisation sampled per iteration — giving you three separate views into resource consumption.

  • Peak heap memory isolated to inference code only, not the whole process
  • RAM RSS tracks total process memory for real deployment estimation
  • CPU average and peak percentage sampled across all iterations
  • All memory metrics shown alongside latency in a single result view

03 / Accuracy Metrics

Speed means nothing
if predictions are wrong.

Upload a ground-truth labels file alongside your test data and ModelBench automatically scores your model's predictions. No extra code required — it detects classifier vs regressor and picks the right metrics automatically.

  • Classifiers: accuracy score + weighted F1 (or binary F1 for two-class models)
  • Regressors: RMSE, MAE, and R² coefficient of determination
  • Auto-detection of task type — no config flags to set
  • Labels file supports .csv and .npy format

04 / Batch Size Sweep

Find your optimal
batch size automatically.

Different batch sizes can produce dramatically different throughput and latency profiles. Rather than running benchmarks manually one by one, the Batch Sweep profiles your model across up to 8 batch sizes in a single submission and highlights the winners.

  • Enter comma-separated batch sizes — e.g. 1,8,32,128,256
  • Separately identifies best batch size for throughput vs best for latency
  • Results table with per-size latency, p99, throughput and memory peak
  • Two auto-rendered charts: throughput bars + latency line over batch size

05 / Model Introspection

Understand your model
without reading its code.

Every benchmark automatically extracts metadata from your model file and surfaces it alongside performance results. No manual inspection, no custom scripts — ModelBench reads the model and tells you what's inside.

  • sklearn: class name, all hyperparameters, n_features, n_classes, is_classifier/regressor
  • Feature importance bar chart for tree-based models (top 10 features)
  • PyTorch: total parameter count · TensorFlow: layers + parameters
  • ONNX: full input/output spec with shapes and dtypes

06 / Async Jobs & History

Run long benchmarks
in the background.

Large models or high iteration counts can take minutes to benchmark. Toggle Async Mode to dispatch the job to a background worker and get a live progress update while it runs. Every result is stored automatically — tag, compare, and export at any time.

  • One-click async toggle — no config, no CLI, no extra setup
  • Live progress bar with percentage updated every 800ms
  • Full history with timestamp, batch size, and key metrics at a glance
  • Tag results with notes, compare any two or more runs side-by-side
  • Export individual results or bulk-export all history to CSV
